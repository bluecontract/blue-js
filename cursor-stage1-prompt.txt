Behavioral requirements that must be preserved
- `DocBuilder.edit(existing)` must mutate and return the same `BlueNode` instance.
- `DocBuilder.from(existing)` must clone before editing and return a different instance.
- `DocBuilder.expr('x')` must return `${x}`.
- `DocBuilder.expr('${x}')` must remain `${x}` and must not double-wrap.
- `buildDocument()` must throw when a section is still open, with a clear message equivalent to Java intent.
- Section tracking must collect related field pointers and related contract keys and materialize a `Conversation/Document Section` contract on `endSection()`.
- `.channel(name, contractLike)` must overwrite or specialize an existing channel contract at the same key.
- Operation implementation contract key is `<operationKey>Impl`.
- Operation builder must support editing an existing operation contract and later adding an implementation contract.
- `.request(requestSchema...)` must accept a request schema already expressed as a `BlueNode` or Blue-shaped object and store it under `contracts/<operation>/request`.
- `.noRequest()` must remove the request from the operation when editing an existing operation.
- `.replaceExpression(...)` must store a wrapped `${...}` expression.
- `.emitType(...)` must create a `Conversation/Trigger Event` step with an event node whose `type` is derived from the provided type input.
- `buildDocument()` should return the built `BlueNode` directly without extra post-processing unless a public runtime helper is required.

Type model for stage 1
For `.type(...)`, field builder `.type(...)`, operation builder `.requestType(...)`, and `emitType(...)`, support these inputs:
- a string type alias such as `'Integer'` or `'Conversation/Event'`
- a `{ blueId: string }` object
- a `BlueNode` representing a type node
- a Zod schema with `typeBlueId` annotation available through public language annotation helpers

For ordinary values and channel/request/event payload shapes in stage 1, support:
- primitives
- arrays
- plain Blue-shaped JSON-like objects
- `BlueNode`

Do not import non-public internals such as `@blue-labs/language/lib/...`.
If schema/type conversion needs a helper, implement a small local helper inside `libs/sdk-dsl` using only public APIs.
